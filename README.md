# Puoppo 🕊

## 🚀 概要：なぜ「Puoppo」なのか

**「現場には、現場の最適解がある。」**

IT資格ゼロ、プログラミング実務未経験のド初心者が、基礎学習からわずか42時間で制作にありついた、AIトレンド自動分析システム――それが『Puoppo（ポッポ）』です。

特定の流行を追う一発芸のようなシステムではなく、「項目さえ変えれば、どんな現場のデータも制御できる」という、長年履き込んでも壊れないデニムのような普遍的な設計思想で骨格を仕立て上げました。

## 💡 特徴と圧倒的なタイパ

- **爆速の自動収集＆要約**
  テキストボックスに「知りたい情報（例：政治・物価高、最新音楽ニュース等）」を打ち込むだけ。AIがインターネット上の最新記事を100件レベルで超高速収集し、わずか10秒で濃厚に要約してPC・スマホに滑り込ませます。無駄なコメンテーターの意見をダラダラ聴く必要はありません。
- **視線誘導を計算したインターフェース設計**
  使う人の迷いを無くし、現場の人間が最もスムーズに、無駄なく動けるための地に足のついたUIを構築。
- **安全なRSSフィードを活用した「C案のルート精神」**
  大手サイトの堅牢なセキュリティの壁に対し、強引に突っ込まず（A案・B案を捨て）、公式の安全な窓口である「RSSフィード」を活用。道路という巨大なシステムの中で周囲と調和し、安全に最短で目的地に達するという物流現場のトラック運転で培った最適化ロジックが、この安全ルートを閃かせました。

## 🛠️ システム環境と修得技術

- **言語・フレームワーク:** Python / Flask (マイクロフレームワーク)
- **データベース:** SQLite3 (CRUD機能の完全制御)
- **フロントエンド:** HTMLレンダリング / Jinja2テンプレート / JavaScript
- **開発環境:** Ubuntu (Lubuntu) / VS Code / .venv (仮想環境)

## 🔧 現場の数に合わせて項目を定義（汎用設計）

FIELDS = {
    "data_01": "売上金額", # 飲食店なら「売上」、物流なら「燃料消費量」
    "data_02": "廃棄数",   # 飲食店なら「廃棄」、学校なら「提出物スコア」
    "data_03": "客数",     # 状況に合わせてラベルを書き換えるだけ
    "data_04": "備考"
}

複雑な修正は不要です。設定ファイルのフィールド定義を書き換えるだけで、システムは瞬時にあらゆる現場（工場の在庫管理、スタッフの勤怠管理など）に適合します。
